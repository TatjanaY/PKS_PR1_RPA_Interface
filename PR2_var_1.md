# ✅ Вариант 1

## 🟢 Задание 1 (базовый уровень)

### Постановка задачи

Создайте пользовательскую форму **«Регистрация пользователя»** для последовательного ввода данных нескольких человек.

### Элементы формы

Добавьте:

1. Текст (строка) — Имя (`name`)
2. Текст (строка) — Email (`email`)
3. Число — Возраст (`age`, min = 14, max = 100)
4. Кнопки:

   * Подтвердить (ОК)
   * Отмена

---

### Подготовка данных

С помощью любого ИИ-сервиса (DeepSeek / ChatGPT) сгенерируйте **10 тестовых записей** вида:

```
Имя, Email, Возраст
```

---

### Требования к алгоритму

1. Форма вызывается **в цикле 10 раз**.
2. На каждой итерации вводится один набор данных.
3. После нажатия ОК данные добавляются в файл `users.txt`.
4. Каждая запись сохраняется с новой строки в формате:

   ```
   Имя;Email;Возраст
   ```
5. При нажатии Отмена ввод прекращается.
6. После завершения вывести сообщение:

   ```
   Сохранено записей: N
   ```

# 🔵 Задание 2 (повышенный уровень)

### Постановка задачи

Создайте форму **«Расширенная регистрация»** с дополнительными элементами, условиями показа и использованием кнопки типа **Триггер**.

---

### Элементы формы

1. Текст — Имя (`name`)
2. Текст — Email (`email`)
3. Число — Возраст (`age`)
4. Чекбокс — Студент (`is_student`)
5. Текст — Учебное заведение (`university`)

   * отображать только если `is_student = Истина`
6. Селект — Тариф (`plan`)

   * варианты: basic, pro, vip
7. Текст (строка) — **Статус проверки** (`status`)

   * только для отображения результата
   * установить `disabled = Истина`
8. Кнопки:

   * Триггер **«Проверить данные»**
   * ОК
   * Отмена

---

### Логика кнопки Триггер

Создайте функцию `check_user_data(form_result)`, которая:

1. Проверяет:

   * email содержит символ `@`
   * возраст ≥ 18
2. Формирует текст:

   * «Данные корректны»
   * или «Ошибка: некорректный email или возраст»
3. Возвращает строку результата.

Результат работы функции должен:

* записываться в ID результата `status_result`
* отображаться в поле `status`.

⚠ Форма не должна закрываться при нажатии Триггера.

---

### Динамические параметры при вызове формы

Программно задать:

* заголовок окна: **Регистрация 🚀**
* изменить цвет фона;
* значение по умолчанию: `plan = basic`.

---

### Подготовка данных

Сгенерируйте с помощью ИИ **10 расширенных записей**:

```
Имя, Email, Возраст, Студент (да/нет), ВУЗ, Тариф
```

---

### Требования к алгоритму

1. Ввод выполняется последовательно для 10 пользователей.
2. Перед подтверждением обязательно нажимать кнопку «Проверить данные».
3. После нажатия ОК данные сохраняются в файл `users_extended.csv`:

```
Имя;Email;Возраст;Студент;ВУЗ;Тариф
```

4. Если форма возвращает пустое значение (Отмена), выполнение прекращается.
